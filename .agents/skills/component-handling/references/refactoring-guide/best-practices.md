# Best Practices

1. **Move when proven, not predicted** — Wait for actual reuse. Don't anticipate "might need".

2. **One step at a time** — Move first, then optimize. Don't combine multiple changes.

3. **Maintain backward compatibility** — Create wrapper if needed. Deprecate gradually.

4. **Document the move** — Add migration notes. Update team docs. Comment on why moved.

5. **Separate git commit** — One commit for the move. Clear message. Easy to revert.
